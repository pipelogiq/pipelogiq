# Security Policy

## Supported Versions

Pipelogiq is in preview (v0.x). Security fixes are applied to the latest release on the `main` branch only.

| Version | Supported |
|---------|-----------|
| 0.x     | Current development line |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email: **security@pipelogiq.dev** (or use [GitHub Security Advisories](https://github.com/pipelogiq/pipelogiq/security/advisories/new) if available).

Include:

- Description of the vulnerability
- Steps to reproduce
- Affected version(s)
- Potential impact

## Response Process

1. We will acknowledge receipt within 5 business days.
2. We will investigate and provide an initial assessment within 10 business days.
3. We will work with you on a fix and coordinate disclosure.

As a preview project, we cannot guarantee specific timelines, but we take security reports seriously and will respond as quickly as possible.

## Scope

The following are in scope:

- Authentication and authorization bypasses
- SQL injection, XSS, or other injection attacks
- Exposure of secrets or credentials
- Privilege escalation

Out of scope:

- Denial of service in local development setups
- Issues requiring physical access
- Social engineering
